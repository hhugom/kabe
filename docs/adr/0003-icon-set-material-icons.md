# Icon set: Material Icons via @expo/vector-icons

The HUD aesthetic (aesthetic-direction.md) calls for filled, chunky glyphs; navigation-surface.md names Material Symbols (filled 500+) or Lucide filled as candidates. We pick Google's Material Icons via `@expo/vector-icons` (`MaterialIcons`): it ships as an Expo peer with zero font-loading setup, all glyphs are filled and chunky by default, and it covers every icon the current archetypes need (chevron-back, three-dot menu, play/pause, close, arrows, tab-bar glyphs). Rejected `lucide-react-native` because Lucide is primarily a stroke set — its filled-glyph coverage is thin, most icons only exist in outline, and forcing a `fill` prop on stroke paths does not produce the chunky HUD weight the palette asks for. Rejected the newer Material Symbols variable font (the doc's first-listed option) because it isn't in `@expo/vector-icons` and would require self-hosting a variable font file on both platforms — cost that outweighs the marginal design win of variable weight when the classic Material set already hits "filled + weight 500+" by construction. Callers go through the `<Icon>` primitive in `mobile/src/components/Icon.tsx`, which owns the choice so a future swap is one file.
